# 本地 OCR 识别优化说明

## 最新更新（2026-03-05 下午）

### 修复 pdf-parse v2 兼容性问题

**问题描述**：
- pdf-parse 升级到 v2 后，API 发生了重大变化
- 原有的函数调用方式 `pdfParse(buffer)` 不再有效
- 导致 PDF 文本提取失败，OCR 无法识别发票信息

**解决方案**：

1. **更新 pdf-parse 导入和使用方式**
   ```typescript
   // 旧版本 (v1)
   const pdfParse = require('pdf-parse')
   const data = await pdfParse(buffer)

   // 新版本 (v2) - 正确方式
   const { PDFParse } = require('pdf-parse')
   const parser = new PDFParse({ data: buffer })
   const result = await parser.getText()
   ```

2. **优化正则表达式匹配**

   **金额提取优化**：
   - 优先匹配大写金额后的小写金额（最准确）：
     ```typescript
     /[壹贰叁肆伍陆柒捌玖拾佰仟万亿圆整角分]+\s*[¥￥]\s*([\d,]+\.\d{2})/
     ```
   - 这个模式能准确识别"价税合计（大写）"后面的金额
   - 示例：在发票中有多个金额时（合计 ¥813.47、税额 ¥38.53、价税合计 ¥852.00），能正确识别价税合计金额 852.00
   - 匹配逻辑：查找"大写金额文字 + ¥ + 数字"的模式，如"捌佰伍拾贰圆整 ¥852.00"

   **发票号码提取优化**：
   - 新增匹配"开票人:"前面的长数字：
     ```typescript
     /开票人[：:][^\d]*(\d{20,24})/
     ```
   - 支持 20-24 位的长发票号码
   - 示例：能正确识别"开票人:\n25117000000382687751"中的发票号码

**测试结果**：
- ✅ 金额：852.00（正确识别价税合计）
- ✅ 日期：2025-04-08（正确识别开票日期）
- ✅ 发票号码：25117000000382687751（正确识别20位发票号码）
- ✅ 报销类型：保险服务（从项目名称 "*保险服务*附加公共场所" 中提取）

### 新增功能：报销类型自动识别

**功能说明**：
- 从发票的项目名称中自动提取报销类型
- 提取规则：匹配第一个 `*` 到第二个 `*` 之间的内容
- 示例：`*保险服务*附加公共场所` → 提取 "保险服务"

**实现细节**：
```typescript
// 正则表达式匹配
const typePattern = /\*([^*]+)\*/
const typeMatch = text.match(typePattern)
if (typeMatch && typeMatch[1]) {
  result.type = typeMatch[1].trim()
}
```

**前端集成**：
- 更新 `OcrResult` 接口，添加 `type` 字段
- 更新 `useInvoice` composable，在处理 OCR 结果时提取报销类型
- 发票表格自动显示识别的报销类型

## 使用建议

### 1. 上传标准电子发票
- ✅ 推荐：电子发票 PDF（含文本层）
- ⚠️ 可能识别失败：扫描件 PDF（纯图片）
- ❌ 不支持：图片格式（JPG、PNG）

### 2. 识别结果验证
上传发票后，系统会自动识别以下信息：
- 报销金额
- 开票日期
- 发票号码
- 发票代码
- 销售方名称
- 购买方名称
- 税额

**请务必人工核对识别结果！**

### 3. 手动修正
如果识别不准确，可以：
1. 在发票明细表中直接修改
2. 点击发票卡片查看详情并修改
3. 删除错误的识别结果，手动输入

## 常见问题

### Q: 为什么有些发票识别不出来？
A: 可能原因：
1. PDF 是扫描件（纯图片，无文本层）
2. PDF 格式特殊或加密
3. 发票格式不标准

解决方案：手动填写发票信息

### Q: 识别的金额不对怎么办？
A:
1. 检查 PDF 是否为电子发票
2. 在发票明细表中手动修改金额
3. 系统会保存您的修改

### Q: 能否提高识别准确率？
A:
1. 确保上传的是标准电子发票
2. 避免上传扫描件或照片
3. 如需更高准确率，可考虑使用云端 OCR（需配置 API）
