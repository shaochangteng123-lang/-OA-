# 本地 OCR 发票识别说明

## 概述

系统已升级为**本地 OCR 识别方案**，所有发票识别在本地完成，数据不会上传到任何外部服务器，确保数据安全。

## 主要特点

### ✅ 优势

1. **数据安全**
   - 所有发票数据在本地处理
   - 不会上传到百度、腾讯等第三方服务器
   - 符合数据安全和隐私保护要求

2. **零成本**
   - 完全免费，无需申请 API 密钥
   - 无调用次数限制
   - 无需支付任何费用

3. **开箱即用**
   - 无需配置任何环境变量
   - 安装后即可使用
   - 维护成本低

### ⚠️ 注意事项

1. **识别准确率**
   - 本地 OCR 准确率可能低于云端服务（百度、腾讯等）
   - 建议用户上传后手动核对识别结果
   - 特别是金额、日期等关键信息

2. **适用范围**
   - 适用于标准格式的电子发票
   - 对于扫描件或特殊格式发票，建议手动填写
   - 支持 PDF 格式发票

## 技术实现

### 使用的技术

- **Tesseract.js**：开源 OCR 引擎
- **pdf-lib**：PDF 文件解析
- **正则表达式**：智能提取发票信息

### 识别流程

```
用户上传 PDF
    ↓
本地解析 PDF 文件
    ↓
提取文本内容
    ↓
正则匹配关键信息
    ↓
返回识别结果
    ↓
用户确认/修改
```

### 可识别的信息

- ✅ 金额（价税合计）
- ✅ 开票日期
- ✅ 发票号码
- ✅ 销售方名称
- ✅ 购买方名称
- ✅ 税额
- ✅ 发票代码

## 使用建议

### 最佳实践

1. **上传标准格式发票**
   - 使用电子发票 PDF 文件
   - 确保 PDF 文件清晰可读
   - 避免上传扫描件或照片

2. **人工核对**
   - 上传后检查识别结果
   - 重点核对金额和日期
   - 必要时手动修改

3. **文件大小**
   - 单个文件不超过 5MB
   - 系统会自动压缩超大文件
   - 建议使用标准大小的 PDF

### 常见问题

**Q: 识别不准确怎么办？**
A: 可以手动修改识别结果，系统会保存您的修改。

**Q: 支持哪些发票类型？**
A: 支持增值税专用发票、增值税普通发票、电子发票等标准格式发票。

**Q: 可以上传图片吗？**
A: 目前只支持 PDF 格式，建议将图片转换为 PDF 后上传。

**Q: 识别速度如何？**
A: 本地识别速度较快，通常在 1-3 秒内完成。

## 升级说明

### 从云端 OCR 迁移

如果您之前使用的是百度或腾讯云 OCR，现在可以：

1. 删除 `.env` 文件中的 OCR 配置：
   ```bash
   # 可以删除这些配置
   # BAIDU_OCR_API_KEY=...
   # BAIDU_OCR_SECRET_KEY=...
   # TENCENT_SECRET_ID=...
   # TENCENT_SECRET_KEY=...
   ```

2. 重启服务：
   ```bash
   docker compose down
   docker compose up -d --build
   ```

3. 系统会自动使用本地 OCR

### 无缝切换

- 无需修改任何代码
- 前端界面保持不变
- 用户体验一致
