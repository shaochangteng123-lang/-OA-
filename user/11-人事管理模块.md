# 11-人事管理模块

## 模块概述

人事管理模块负责员工全生命周期管理，包括入职、转正、离职、请假等人事流程的申请、审批和数据管理。

## 主要功能

### 1. 入职管理
- 新建入职申请
- 入职申请审批
- 办理入职手续
- 入职信息查看和编辑

### 2. 转正管理

#### 用户端功能（转正申请页面）
- 查看实习期状态（入职日期、试用期截止、剩余天数）
- 下载转正申请表模板
- 上传填写完成的转正申请书
- 提交转正申请
- 查看审批结果

#### 管理员端功能（转正管理页面）
- 上传/管理转正申请表模板
- 查看待转正员工列表
- 查看员工上传的转正申请书
- 审批转正申请（通过/拒绝）
- 转正统计数据展示

### 3. 离职管理

#### 用户端功能（离职页面）
- 下载管理员上传的离职申请表模板与交接单模板
- 选择交接人并保存离职申请草稿
- 上传离职申请表、离职人交接单及按离职原因要求的补充材料
- 提交离职申请
- 查看审批流程、时间线与审批意见
- 作为交接人处理他人的交接任务，上传交接单并确认完成

#### 管理员端功能（员工数据 > 离职管理）
- 上传/管理离职申请表模板
- 上传/管理交接单模板
- 上传/管理终止/解除劳动关系证明、固定资产交接单、离职经济补偿协议书、离职其他费用结算约定、合伙人离任分红结算模板
- 查看离职申请列表与状态筛选
- 查看双方上传的交接单、离职申请表及补充材料
- 审批离职申请（通过/驳回）
- 审批通过后将员工状态更新为已离职

### 4. 请假管理
- 新建请假申请
- 请假审批
- 请假类型管理（年假、事假、病假、婚假、产假、陪产假）
- 请假记录查询

### 5. 员工数据管理
- 员工信息查看和编辑
- 员工统计数据展示
- 员工数据导出
- 员工状态管理（在职、试用期、已离职）

## 入职流程强制引导

新用户首次登录系统时，会被强制引导到入职页面填写员工基础信息。只有提交员工信息后，才能正常使用系统其他功能。

### 流程说明

1. **首次登录**: 用户登录后，系统检查其员工信息状态
2. **未填写/草稿状态**: 自动跳转到入职页面 (`/onboarding`)
3. **填写信息**: 用户填写员工基础信息并保存草稿
4. **提交信息**: 用户提交信息后，状态变为 `submitted`
5. **正常使用**: 提交后可正常访问系统其他功能

### 状态检查逻辑

- `none`: 没有员工信息记录 → 跳转入职页面
- `draft`: 草稿状态 → 跳转入职页面
- `submitted`: 已提交 → 正常访问系统

- [src/stores/auth.ts](../src/stores/auth.ts) - `profileStatus` 和 `hasCompletedOnboarding` 状态

## 页面功能详解

### 离职管理页面

**路由**: `/resignation`

**主要功能**:
- 下载管理员上传的离职申请表模板与交接单模板
- 填写离职类型、离职日期、离职原因、交接人
- 保存离职申请草稿
- 上传离职申请表、离职人交接单及按离职原因要求的补充材料
- 提交离职申请并等待交接人处理
- 查看双方确认状态、审批进度和时间线
- 作为交接人处理待我处理的离职交接任务
- 离职日期需至少提前 1 个月申请，1 个月内日期不可选
- 主动离职/合同到期需补充上传：终止/解除劳动关系证明、固定资产交接单、离职其他费用结算约定
- 被辞退额外需补充上传：离职经济补偿协议书

**状态流转**:
```
draft → submitted → handover_confirmed → mutual_confirmed → approved
                               └────────────→ rejected
```

### 员工数据页面 - 离职管理 Tab

**主要功能**:
- 上传和删除离职申请表模板、交接单模板
- 上传和删除终止/解除劳动关系证明、固定资产交接单、离职经济补偿协议书、离职其他费用结算约定、合伙人离任分红结算模板
- 按状态筛选离职申请列表
- 查看离职人和交接人上传的附件
- 在详情弹窗中按材料类型查看必传材料、缺失状态和已上传文件
- 填写审批意见并通过/驳回离职申请
- 审批通过后同步将员工状态更新为 `resigned`

### 员工数据页面 - 离职档案 Tab

**主要功能**:
- 在员工详情弹窗中于“人事档案”后新增“离职档案”Tab
- 聚合展示员工离职类型、离职日期、当前状态、提交时间、审批时间、审批意见、交接人
- 展示离职人上传的全部离职材料
- 单独展示交接人上传的工作交接单
- 复用离职附件下载接口进行查看与下载

**相关接口**:
- `GET /api/employees/:id/resignation-archive` - 获取员工离职档案聚合数据
- `GET /api/resignation/requests/:id/documents/:docId/download` - 下载/预览离职附件

**展示规则**:
- 无离职记录时显示“该员工暂无离职申请记录”
- 若员工状态已是 `resigned` 但没有离职申请记录，则按员工状态兜底展示基础离职档案
- 工作交接单对应 `handover_form_handover` 类型附件

### 转正申请页面（用户端）

**路由**: `/probation`

**主要功能**:
- **实习期状态卡片**: 显示入职日期、试用期截止日期、剩余天数（按入职时间到试用期截止时间的区间计算）
- **转正申请表** Tab:
  - 显示管理员上传的转正申请表模板
  - 支持预览和下载模板
- **上传申请书** Tab:
  - 上传填写完成的转正申请书（支持 PDF、JPG、PNG、DOC、DOCX）
  - 上传时可填写转正申请说明
  - 在转正申请表列直接显示已上传文件和上传入口
  - 删除已上传的文件（审批通过前）
  - 提交转正申请
- **流程操作**:
  - 转正前默认状态显示为“实习期”
  - 转正详情和审批流程中展示申请说明
  - 已提交状态可查看审批流程、撤回申请
  - 已驳回状态可查看驳回原因并重新上传后再次提交
  - 驳回后不能撤回，只能删除记录
  - 仅存在真实转正记录且满足“驳回”或“撤回后 pending 且有历史提交记录”时显示“删除记录”按钮
  - 删除记录调用 `DELETE /api/probation/my-record`，会清空当前转正记录、审批流和关联文件，便于重新测试流程

**状态流转**:
```
待提交 (pending) → 待审批 (submitted) → 已通过 (approved)
                                      → 已拒绝 (rejected) → 可重新提交
```

**API 接口**:
- `GET /api/probation/my-status` - 获取当前用户的转正状态，返回 `hasRealConfirmation` 区分真实记录和试用期虚拟 pending 状态，返回 `hasHistory` 标识是否有提交历史
- `GET /api/probation/templates` - 获取转正模板列表
- `POST /api/probation/upload-doc` - 上传转正申请书
- `DELETE /api/probation/my-doc/:docId` - 删除转正申请书
- `GET /api/probation/my-doc/:docId/download` - 下载/预览转正申请书
- `POST /api/probation/apply` - 提交转正申请
- `DELETE /api/probation/my-record` - 删除当前用户的转正记录，清理转正文件、审批实例、审批记录和转正主记录

## 收款信息同步

### 功能说明

付款审批中心的收款人信息与入职填写的基础信息中的收款信息保持一致。用户在入职页面填写收款信息后，管理员在付款页面可以直接看到该用户的收款信息。

### 数据来源

收款人信息优先从 `employee_profiles` 表获取（入职时填写的信息），如果入职信息中没有收款信息，则回退到 `users` 表（兼容旧数据）。

### 相关 API

- `GET /api/approval/payee-info/:userId` - 获取收款人信息
  - 优先从 `employee_profiles` 表获取
  - 回退到 `users` 表（兼容旧数据）

### 收款信息字段

| 字段 | 说明 |
|------|------|
| bank_account_name | 收款人姓名 |
| bank_account_phone | 收款人手机号 |
| bank_name | 开户行 |
| bank_account_number | 银行卡号 |
