# 11-人事管理模块

## 模块概述

人事管理模块负责员工全生命周期管理，包括入职、转正、离职、请假等人事流程的申请、审批和数据管理。

## 主要功能

### 1. 入职管理
- 新建入职申请
- 入职申请审批
- 办理入职手续
- 入职信息查看和编辑

### 2. 转正管理

#### 用户端功能（转正申请页面）
- 查看实习期状态（入职日期、试用期截止、剩余天数）
- 下载转正申请表模板
- 上传填写完成的转正申请书
- 提交转正申请
- 查看审批结果

#### 管理员端功能（转正管理页面）
- 上传/管理转正申请表模板
- 查看待转正员工列表
- 查看员工上传的转正申请书
- 审批转正申请（通过/拒绝）
- 转正统计数据展示

### 3. 离职管理
- 新建离职申请
- 离职申请审批
- 离职交接办理
- 离职类型管理（主动离职、合同到期、辞退）

### 4. 请假管理
- 新建请假申请
- 请假审批
- 请假类型管理（年假、事假、病假、婚假、产假、陪产假）
- 请假记录查询

### 5. 员工数据管理
- 员工信息查看和编辑
- 员工统计数据展示
- 员工数据导出
- 员工状态管理（在职、试用期、已离职）

## 入职流程强制引导

新用户首次登录系统时，会被强制引导到入职页面填写员工基础信息。只有提交员工信息后，才能正常使用系统其他功能。

### 流程说明

1. **首次登录**: 用户登录后，系统检查其员工信息状态
2. **未填写/草稿状态**: 自动跳转到入职页面 (`/onboarding`)
3. **填写信息**: 用户填写员工基础信息并保存草稿
4. **提交信息**: 用户提交信息后，状态变为 `submitted`
5. **正常使用**: 提交后可正常访问系统其他功能

### 状态检查逻辑

- `none`: 没有员工信息记录 → 跳转入职页面
- `draft`: 草稿状态 → 跳转入职页面
- `submitted`: 已提交 → 正常访问系统

- [src/stores/auth.ts](../src/stores/auth.ts) - `profileStatus` 和 `hasCompletedOnboarding` 状态

## 页面功能详解

### 入职管理页面

**路由**: `/onboarding`

**主要功能**:
- **员工基础信息** Tab:
  - 填写个人基础信息（姓名、性别、出生日期、身份证号、民族、籍贯、婚姻状况）
  - 填写联系方式（手机号码、电子邮箱、现居住地址）

  - 填写教育经历（学历、毕业院校、专业）

- **入职文件** Tab:
  - 入职所需文件清单展示
  - 文件预览和下载

**状态流转**:
```
草稿 (draft) → 已提交 (submitted)
```

**API 接口**:
- `GET /api/employees/my-profile` - 获取当前用户的员工信息
- `POST /api/employees/my-profile` - 保存/更新员工信息（草稿）
- `POST /api/employees/my-profile/submit` - 提交员工信息
- `GET /api/employees/statistics` - 获取员工统计数据（管理员）

### 转正申请页面（用户端）

**路由**: `/probation`

**主要功能**:
- **实习期状态卡片**: 显示入职日期、试用期截止日期、剩余天数
- **转正申请表** Tab:
  - 显示管理员上传的转正申请表模板

## 权限控制

建议的权限配置：

- **普通员工**: 可以提交自己的请假申请，查看自己的人事记录
- **部门主管**: 可以审批本部门的请假申请
- **HR**: 可以管理所有入职、转正、离职流程，查看所有员工数据
- **管理员**: 拥有所有权限

## 注意事项

1. **日期计算**: 请假天数需要考虑工作日和节假日
2. **试用期计算**: 转正日期需要根据入职日期和试用期长度自动计算
3. **数据一致性**: 员工状态需要与入职、转正、离职流程保持同步
4. **审批权限**: 需要根据组织架构配置审批流程
5. **数据归档**: 离职员工数据需要归档保存，不能删除

## 收款信息同步

### 功能说明

付款审批中心的收款人信息与入职填写的基础信息中的收款信息保持一致。用户在入职页面填写收款信息后，管理员在付款页面可以直接看到该用户的收款信息。

### 数据来源

收款人信息优先从 `employee_profiles` 表获取（入职时填写的信息），如果入职信息中没有收款信息，则回退到 `users` 表（兼容旧数据）。

### 相关 API

- `GET /api/approval/payee-info/:userId` - 获取收款人信息
  - 优先从 `employee_profiles` 表获取
  - 回退到 `users` 表（兼容旧数据）

### 收款信息字段

| 字段 | 说明 |
|------|------|
| bank_account_name | 收款人姓名 |
| bank_account_phone | 收款人手机号 |
| bank_name | 开户行 |
