# 02 - 日历模块

## 模块概述

日历模块提供日程管理功能，支持多种视图、循环事件、事件库管理和节假日显示。

## 主要功能

### 1. 日历视图
- **周视图**：显示一周的日程安排
- **月视图**：月度日程总览
- **日视图**：单日详细日程

### 2. 事件管理
- 创建、编辑、删除日程事件
- 事件拖拽调整时间
- 事件颜色标签分类
- 事件详情查看

### 3. 循环事件
- 使用 rrule 库实现循环规则
- 支持每日、每周、每月、每年重复
- 自定义循环结束条件

### 4. 事件库
- **事件模板**：预设可重用的事件模板
- **事件预设**：针对项目类型的事件集合
- **事件分组**：按类别组织事件
- **政府部门关联**：事件可关联政府部门

### 5. 节假日显示
- 中国法定节假日标注
- 调休工作日标识
- 节假日数据可通过脚本更新

## 数据模型

### CalendarEvent（日程事件）
```typescript
{
  id: string
  title: string
  description?: string
  start_time: string
  end_time: string
  all_day: boolean
  color?: string
  recurrence_rule?: string  // rrule 格式
  created_by: number
  created_at: string
  updated_at: string
}
```

### EventLibrary（事件库）
```typescript
{
  id: number
  name: string
  description?: string
  default_duration?: number
  color?: string
  category?: string
  government_dept_id?: number
}
```

### Holiday（节假日）
```typescript
{
  id: number
  date: string
  name: string
  is_workday: boolean  // true 表示调休工作日
}
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/calendar/events` | GET | 获取日程列表 |
| `/api/calendar/events` | POST | 创建日程 |
| `/api/calendar/events/:id` | PUT | 更新日程 |
| `/api/calendar/events/:id` | DELETE | 删除日程 |
| `/api/calendar/library` | GET | 获取事件库 |
| `/api/calendar/holidays` | GET | 获取节假日数据 |

## 使用示例

### 创建循环事件
```typescript
import { RRule } from 'rrule'

// 每周一上午9点开会
const rule = new RRule({
  freq: RRule.WEEKLY,
  byweekday: [RRule.MO],
  dtstart: new Date('2026-01-20T09:00:00'),
  until: new Date('2026-12-31')
})

const event = {
  title: '周例会',
  start_time: '2026-01-20T09:00:00',
  end_time: '2026-01-20T10:00:00',
  recurrence_rule: rule.toString(),
  color: 'blue'
}
```

### 更新节假日数据
```bash
# 编辑节假日脚本
vi server/scripts/update-holidays.ts

# 运行更新命令
npm run update:holidays
```

## 特色功能

### 1. 事件拖拽
- 支持在日历视图中拖拽调整事件时间
- 自动计算新的开始和结束时间
- 实时保存到后端

### 2. 颜色标签
- 预设多种颜色标签
- 用于分类和快速识别事件类型
- 支持自定义颜色

### 3. 事件预设
- 根据项目类型快速批量创建事件
- 适用于重复性工作流程
- 减少手动创建工作量

## 注意事项

1. **循环事件编辑**：
   - 编辑循环事件时可选择修改单次或全部
   - 删除循环事件会影响所有未来实例

2. **时区处理**：
   - 所有时间统一使用 ISO 8601 格式
   - 前端自动转换为本地时区显示

3. **性能优化**：
   - 日历视图仅加载可见时间范围的事件
   - 使用虚拟滚动优化长列表渲染
