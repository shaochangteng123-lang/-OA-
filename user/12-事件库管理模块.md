# 12-事件库管理模块

## 模块概述

事件库管理模块负责管理项目流程中的事件模板、预设方案和板块配置。通过事件库可以快速创建标准化的项目流程，提高项目管理效率。

## 主要功能

### 1. 事件库管理
- 事件模板的创建、编辑、删除
- 事件类型分类（审批事项、备案事项、其他）
- 事件标准时长设置
- 事件搜索和筛选

### 2. 预设方案管理
- 预设方案的创建、编辑、删除
- 按实施类型和项目类型配置预设
- 板块组合配置
- 默认预设标记

### 3. 板块设置
- 板块的创建、编辑、删除
- 板块分类管理
- 板块包含的事件配置
- 板块标准耗时设置

## 数据模型

### 事件模板 (EventLibraryItem)
```typescript
interface EventLibraryItem {
  id: string                    // 事件ID
  name: string                  // 事件名称
  eventType: 'approval' | 'filing' | 'other'  // 事件类型
  standardDuration?: number     // 标准时长（天）
  description?: string          // 事件描述
  departmentId?: string         // 关联政府部门ID
  createdAt?: string           // 创建时间
  updatedAt?: string           // 更新时间
}
```

**事件类型说明**:
- `approval`: 审批事项 - 需要政府部门审批的事项
- `filing`: 备案事项 - 需要向政府部门备案的事项
- `other`: 其他 - 其他类型的事项

### 板块 (EventBlock)
```typescript
interface EventBlock {
  id: string                    // 板块ID
  name: string                  // 板块名称
  categoryId: string            // 分类ID
  categoryName?: string         // 分类名称
  description?: string          // 板块描述
  events?: EventLibraryItem[]   // 包含的事件列表
  standardDuration?: number     // 标准耗时（天）
  sortOrder?: number            // 排序顺序
  createdAt?: string           // 创建时间
  updatedAt?: string           // 更新时间
}
```

### 板块分类 (BlockCategoryEntity)
```typescript
interface BlockCategoryEntity {
  id: string                    // 分类ID
  name: string                  // 分类名称
  description?: string          // 分类描述
  sortOrder?: number            // 排序顺序
}
```

**默认分类**:
- `cat1`: 前期手续
- `cat2`: 中期审批
- `cat3`: 后期验收

### 预设方案 (EventPreset)
```typescript
interface EventPreset {
  id: string                    // 预设ID
  name: string                  // 预设名称
  implementationType?: string   // 实施类型（新改扩建、历史遗留）
  projectType?: string          // 项目类型（场站、线性、工民建）
  blocks: PresetBlockRef[]      // 板块引用列表
  isDefault?: boolean           // 是否为默认预设
  createdAt?: string           // 创建时间
  updatedAt?: string           // 更新时间
}
```

### 预设板块引用 (PresetBlockRef)
```typescript
interface PresetBlockRef {
  blockId: string               // 板块ID
  sortOrder: number             // 排序顺序
  customName?: string           // 自定义名称
}
```

## 页面功能详解

### 事件库管理页面

**路由**: `/events`

**主要功能**:
- 事件列表展示（表格形式）
- 按事件名称搜索
- 按事件类型筛选
- 新建事件
- 编辑事件
- 删除事件

**表单字段**:
- 事件名称（必填）
- 事件类型（必填）：审批事项、备案事项、其他
- 标准时长（可选）：单位为天
- 描述（可选）

### 板块设置页面

**路由**: `/blocks`

**主要功能**:
- 板块列表展示（表格形式）
- 显示板块名称、分类、包含事件数
- 新建板块
- 编辑板块
- 删除板块

**表单字段**:
- 板块名称（必填）
- 分类（必填）：前期手续、中期审批、后期验收
- 描述（可选）

**板块分类**:
- `cat1`: 前期手续 - 项目启动前需要完成的手续
- `cat2`: 中期审批 - 项目实施过程中的审批事项
- `cat3`: 后期验收 - 项目完成后的验收事项

### 预设方案管理页面

**路由**: `/presets`

**主要功能**:
- 预设方案卡片展示（网格布局）
- 显示实施类型、项目类型、板块数、事件数
- 新建预设方案
- 编辑预设方案
- 删除预设方案（默认预设不可删除）
- 板块配置（添加、排序、移除）

**表单字段**:
- 预设名称（必填）
- 实施类型（必填）：新改扩建、历史遗留
- 项目类型（必填）：场站、线性、工民建
- 板块配置（必填）：至少添加一个板块

**板块配置功能**:
- 从板块库中选择板块
- 支持多选板块
- 板块排序（上移、下移）
- 移除板块
- 显示每个板块的事件数和标准耗时

**实施类型**:
- `新改扩建`: 新建、改建、扩建项目
- `历史遗留`: 历史遗留问题项目

**项目类型**:
- `场站`: 场站类项目（如变电站、泵站等）
- `线性`: 线性工程项目（如管线、道路等）
- `工民建`: 工业与民用建筑项目

## 业务流程

### 事件库使用流程

```
1. 创建事件模板
   ↓
2. 将事件添加到板块
   ↓
3. 创建预设方案，组合多个板块
   ↓
4. 在项目中应用预设方案
   ↓
5. 自动生成项目日程事件
```

### 预设方案配置流程

```
1. 新建预设方案
   ↓
2. 选择实施类型和项目类型
   ↓
3. 从板块库中选择需要的板块
   ↓
4. 调整板块顺序
   ↓
5. 保存预设方案
   ↓
6. 在项目创建时选择该预设
```

## 数据关联

### 事件 → 板块 → 预设 → 项目

```
EventLibraryItem (事件模板)
    ↓ 多对多
EventBlock (板块)
    ↓ 多对多 (通过 PresetBlockRef)
EventPreset (预设方案)
    ↓ 应用到
Project (项目)
    ↓ 生成
CalendarEvent (日程事件)
```

### 关联说明

1. **事件与板块**: 一个板块可以包含多个事件，一个事件可以属于多个板块
2. **板块与预设**: 一个预设可以包含多个板块，一个板块可以被多个预设引用
3. **预设与项目**: 创建项目时可以选择预设方案，自动生成项目流程
4. **项目与日程**: 应用预设后，系统根据板块和事件自动创建日程事件

## 权限控制

建议的权限配置：

- **普通用户**: 只读权限，可以查看事件库、板块和预设
- **项目管理员**: 可以创建和编辑预设方案
- **系统管理员**: 拥有所有权限，可以管理事件库、板块和预设

## 注意事项

1. **默认预设保护**: 默认预设不能删除，只能编辑板块配置
2. **板块顺序**: 预设中的板块顺序会影响项目流程的执行顺序
3. **事件时长**: 标准时长用于项目进度估算，应根据实际情况设置
4. **分类管理**: 板块分类应保持稳定，避免频繁修改
5. **数据一致性**: 删除事件或板块前应检查是否被预设引用

## 数据库表

### event_library - 事件库表
```sql
CREATE TABLE event_library (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  event_type TEXT NOT NULL,
  standard_duration INTEGER,
  description TEXT,
  department_id TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### event_blocks - 板块表
```sql
CREATE TABLE event_blocks (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  category_id TEXT NOT NULL,
  description TEXT,
  standard_duration INTEGER,
  sort_order INTEGER DEFAULT 0,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### block_events - 板块事件关联表
```sql
CREATE TABLE block_events (
  block_id TEXT NOT NULL,
  event_id TEXT NOT NULL,
  sort_order INTEGER DEFAULT 0,
  PRIMARY KEY (block_id, event_id)
);
```

### event_presets - 预设方案表
```sql
CREATE TABLE event_presets (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  implementation_type TEXT,
  project_type TEXT,
  is_default INTEGER DEFAULT 0,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

CREATE TABLE preset_blocks (

CREATE TABLE block_categories (

## 使用示例

### 创建完整的项目流程预设

1. **创建事件模板**:
   - 立项批复（审批事项，30天）
   - 环评审批（审批事项，60天）
   - 施工许可（审批事项，20天）
   - 竣工验收（审批事项，30天）

2. **创建板块**:
   - 前期手续板块：包含立项批复、环评审批
   - 施工阶段板块：包含施工许可
   - 验收阶段板块：包含竣工验收

3. **创建预设方案**:
   - 名称：场站类新建项目标准流程
   - 实施类型：新改扩建
   - 项目类型：场站
   - 板块：前期手续 → 施工阶段 → 验收阶段

4. **应用到项目**:
   - 创建新项目时选择该预设
   - 系统自动生成所有事件到项目日程
   - 根据标准时长自动计算预计完成时间
