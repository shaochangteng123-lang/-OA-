# 04 - 项目管理模块

## 模块概述

项目管理模块用于管理北京基础设施建设项目的全生命周期，包括立项、进度跟踪、归档和结算。

## 主要功能

### 1. 项目生命周期管理
- **立项阶段**：项目初始信息录入
- **进度跟踪**：项目执行过程监控
- **归档管理**：已完成项目存档
- **结算流程**：项目财务结算

### 2. 项目信息管理
- 项目基本信息（名称、编号、类型）
- 项目状态跟踪
- 所属区域管理
- 联系人信息
- 项目文档附件

### 3. 项目分类
- 按项目类型分类
- 按所属区域分类
- 按项目状态分类
- 自定义标签分类

### 4. 项目搜索与筛选
- 按项目名称搜索
- 按状态筛选
- 按区域筛选
- 按时间范围筛选

## 数据模型

### Project（项目）
```typescript
{
  id: number
  name: string                    // 项目名称
  project_code?: string           // 项目编号
  type?: string                   // 项目类型
  status: string                  // 项目状态
  district?: string               // 所属区域
  description?: string            // 项目描述
  start_date?: string             // 开始日期
  end_date?: string               // 结束日期
  contact_person?: string         // 联系人
  contact_phone?: string          // 联系电话
  budget?: number                 // 项目预算
  actual_cost?: number            // 实际成本
  created_by: number              // 创建人
  created_at: string
  updated_at: string
}
```

### 项目状态
```typescript
enum ProjectStatus {
  PLANNING = '规划中',
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/projects` | GET | 获取项目列表 |
| `/api/projects/:id` | GET | 获取项目详情 |
| `/api/projects` | POST | 创建项目 |
| `/api/projects/:id` | PUT | 更新项目 |
| `/api/projects/:id` | DELETE | 删除项目 |
| `/api/projects/:id/status` | PUT | 更新项目状态 |
| `/api/projects/:id/attachments` | POST | 上传项目附件 |

## 使用示例

### 创建新项目
```typescript
import { api } from '@/utils/api'

const createProject = async () => {
  const projectData = {
    name: '朝阳区基础设施改造项目',
    project_code: 'BJ-CY-2026-001',
    type: '基础设施',
    status: 'APPROVED',
    district: '朝阳区',
    description: '改造区域内老旧基础设施',
    start_date: '2026-02-01',
    budget: 5000000,
    contact_person: '张三',
    contact_phone: '13800138000'
  }

  await api.post('/api/projects', projectData)
}
```

### 更新项目状态
```typescript
const updateProjectStatus = async (projectId: number, status: string) => {
  await api.put(`/api/projects/${projectId}/status`, { status })
}
```

### 项目搜索
```typescript
const searchProjects = async (filters: any) => {
  const params = new URLSearchParams()

  if (filters.name) params.append('name', filters.name)
  if (filters.status) params.append('status', filters.status)
  if (filters.district) params.append('district', filters.district)

  const response = await api.get(`/api/projects?${params}`)
  return response.data
}
```

## 注意事项

1. **状态流转规则**：
   - 项目状态变更需遵循业务流程
   - 某些状态变更需要权限审批
   - 已归档项目不可直接修改

2. **数据完整性**：
   - 必填字段需要验证
   - 日期逻辑需要校验（结束日期不能早于开始日期）
   - 预算和成本需要数值验证

3. **文档管理**：
   - 附件大小限制
   - 支持的文件格式
   - 附件存储路径规划
