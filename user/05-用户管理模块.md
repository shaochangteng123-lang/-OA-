# 05 - 用户管理模块

## 模块概述

用户管理模块用于管理系统用户信息、角色分配和权限设置，支持用户查询、编辑和权限管理。

## 主要功能

### 1. 用户列表管理
- 查看所有系统用户
- 用户信息展示（姓名、邮箱、角色、状态）
- 用户搜索和筛选
- 用户状态管理（启用/禁用）

### 2. 角色管理
- 分配用户角色
- 角色权限查看
- 批量角色分配

### 3. 用户信息编辑
- 更新用户基本信息
- 修改用户角色
- 重置用户密码（如适用）

### 4. 用户活动监控
- 查看用户登录历史
- 用户操作日志
- 活跃度统计

## 数据模型

### User（用户）
```typescript
{
  id: string
  username: string | null           // 登录用户名
  password_hash: string | null      // 密码哈希
  name: string                      // 显示名称
  email?: string                    // 邮箱
  mobile?: string                   // 手机号
  avatar_url?: string               // 头像 URL
  role: 'super_admin' | 'admin' | 'general_manager' | 'user' | 'guest'  // 用户角色
  status: 'active' | 'inactive'     // 用户状态
  department?: string               // 部门
  position?: string                 // 职位
  last_login_at?: string            // 最后登录时间
  created_at: string
  updated_at: string
}
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/users` | GET | 获取用户列表 |
| `/api/users` | POST | 更新用户信息 |
| `/api/users/create` | POST | 创建新用户 |
| `/api/users/:id` | DELETE | 删除用户 |
| `/api/users/:id/reset-password` | POST | 重置用户密码 |

## 使用示例

### 获取用户列表
```typescript
import { api } from '@/utils/api'

const fetchUsers = async (filters?: any) => {
  const params = new URLSearchParams()

  if (filters?.role) params.append('role', filters.role)
  if (filters?.is_active !== undefined) {
    params.append('is_active', String(filters.is_active))
  }
  if (filters?.search) params.append('search', filters.search)

  const response = await api.get(`/api/users?${params}`)
  return response.data
}
```

### 更新用户角色
```typescript
const updateUserRole = async (userId: number, role: string) => {
  await api.put(`/api/users/${userId}/role`, { role })
}
```

### 禁用用户
```typescript
const disableUser = async (userId: number) => {
  await api.put(`/api/users/${userId}/status`, { is_active: false })
}
```

## 角色权限对照表

| 角色 | 权限描述 |
|------|----------|
| `super_admin` | 所有权限，包括系统配置 |
| `admin` | 管理用户、项目、日程等业务数据 |
| `general_manager` | 总经理，负责商务报销审批 |
| `user` | 查看和编辑自己的数据，查看公共数据 |
| `guest` | 仅限查看权限 |

详细权限矩阵参见 [src/types/index.ts](../src/types/index.ts) 中的 `ROLE_PERMISSIONS`。

## 初始管理员创建

### 命令行工具
```bash
npm run create:admin
```

按提示输入：
- 用户名（仅字母、数字、下划线，至少3个字符）
- 密码（至少6个字符）
- 显示名称

### 注意事项
- 首次部署必须通过命令行创建初始管理员
- 后续用户由管理员通过用户管理页面创建
- 角色存储在数据库中，可通过管理界面修改

## 创建用户规则

### 必填字段
| 字段 | 要求 | 说明 |
|------|------|------|
| 用户名 | 必填，至少2个字符 | 使用员工姓名，用户名即为显示名称 |
| 密码 | 必填，至少6个字符 | 登录密码 |
| 手机号 | 必填，必须11位 | 如不是11位提示"手机号格式不正确" |
| 部门 | 必填 | 员工所属部门 |
| 职位 | 必填 | 员工职位 |

### 选填字段
| 字段 | 要求 | 说明 |
|------|------|------|
| 邮箱 | 选填，需有效格式 | 输入有效邮箱地址 |
| 角色 | 选填，默认为普通用户 | admin/general_manager/user/guest |

### 验证规则
- **用户名**：不能重复，至少2个字符
- **密码**：至少6个字符
- **手机号**：必须是11位数字，以1开头，第二位为3-9
- **邮箱**：如填写需符合邮箱格式
- **手机号唯一性**：同一手机号不能重复注册

## 注意事项

1. **角色变更**：
   - 角色变更立即在数据库生效
   - 用户需重新登录才能获得新权限
   - 降低用户权限需谨慎操作

2. **用户禁用**：
   - 禁用用户会立即阻止其登录
   - 已登录的会话可能仍然有效（直到过期）
   - 重要用户禁用前需要确认

3. **数据安全**：
   - 密码使用 bcrypt 加密存储
   - 操作日志完整记录
   - 定期备份用户数据

4. **权限要求**：
   - 创建/删除用户需要 super_admin 权限
   - 编辑用户信息需要 admin 及以上权限
   - 重置密码需要 super_admin 权限
