# 03 - 工作日志模块

## 模块概述

工作日志模块提供基于 TipTap 的富文本编辑器，支持 Notion 风格的编辑体验，用于记录每日工作内容。

## 主要功能

### 1. 富文本编辑
- **Notion 风格编辑器**：类似 Notion 的编辑体验
- **丰富的格式支持**：
  - 标题（H1-H6）
  - 粗体、斜体、下划线
  - 代码块（支持语法高亮）
  - 任务列表（待办事项）
  - 链接、高亮
  - 文本颜色
  - 文本对齐

### 2. 自动保存
- 编辑内容自动保存为草稿
- 每个用户每天一条草稿记录
- 防止意外关闭导致内容丢失

### 3. 日志管理
- 按日期查看历史日志
- 支持搜索和筛选
- 导出日志内容

### 4. 一日一志
- 每个用户每天只能有一条工作日志
- 当天可多次编辑
- 次日自动创建新日志

## 数据模型

### Worklog（工作日志）
```typescript
{
  id: number
  user_id: number
  log_date: string        // 日志日期（YYYY-MM-DD）
  content: string         // JSON 格式的 TipTap 内容
  created_at: string
  updated_at: string
}
```

### Draft（草稿）
```typescript
{
  id: number
  user_id: number
  draft_date: string      // 草稿日期
  content: string         // JSON 格式的 TipTap 内容
  updated_at: string
}
```

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/worklogs` | GET | 获取工作日志列表 |
| `/api/worklogs/:date` | GET | 获取指定日期的日志 |
| `/api/worklogs` | POST | 创建工作日志 |
| `/api/worklogs/:id` | PUT | 更新工作日志 |
| `/api/worklogs/:id` | DELETE | 删除工作日志 |
| `/api/drafts/:date` | GET | 获取指定日期的草稿 |
| `/api/drafts/:date` | POST | 保存草稿 |


### 保存日志
```typescript
import { api } from '@/utils/api'

// 保存今日工作日志
const saveWorklog = async (content: any) => {
  const today = new Date().toISOString().split('T')[0]

  await api.post('/api/worklogs', {
    log_date: today,
    content: JSON.stringify(content)
  })
}
```

## 特色功能

### 1. Slash 命令
- 输入 `/` 触发命令菜单
- 快速插入标题、列表、代码块等
- 类似 Notion 的交互体验

### 2. Markdown 快捷输入
- 支持 Markdown 语法快捷输入
- 自动转换为富文本格式
- 提升编辑效率

### 3. 拖拽排序
- 支持内容块拖拽排序
- 可视化调整内容结构

### 4. 实时协作（未来）
- 预留多人实时编辑能力
- 支持内容冲突解决

## 注意事项

1. **内容格式**：
   - 日志内容以 JSON 格式存储
   - 保存前确保内容有效性
   - 避免存储过大的图片（建议使用外链）

2. **自动保存策略**：
   - 编辑停止 2 秒后触发自动保存
   - 草稿和正式日志分开存储
   - 草稿不会覆盖已保存的日志

3. **性能优化**：
   - 长文档使用延迟加载
   - 代码高亮使用 Web Worker
   - 避免频繁重新渲染
