# 01 - 认证模块

## 模块概述

认证模块负责用户身份验证和权限管理，使用飞书 OAuth 2.0 进行第三方登录。

## 主要功能

### 1. 飞书 OAuth 登录
- 用户点击登录按钮跳转到飞书授权页面
- 飞书回调后获取用户信息
- 自动创建或更新用户记录

### 2. 会话管理
- 基于 express-session 的会话存储
- 会话数据保存在 SQLite 数据库
- 支持持久化登录状态

### 3. 角色权限系统
- **四种角色**：
  - `super_admin` - 超级管理员
  - `admin` - 管理员
  - `user` - 普通用户
  - `guest` - 访客

- **权限矩阵**：定义在 [src/types/index.ts](../src/types/index.ts)
- **权限检查**：通过 `authStore.hasPermission()` 方法

### 4. 管理员配置
- 配置文件：[server/config/admins.json](../server/config/admins.json)
- 通过飞书用户名匹配分配管理员角色
- 配置修改后下次登录生效（无需重启服务）

## API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/auth/feishu` | GET | 获取飞书登录 URL |
| `/api/auth/feishu/callback` | GET | 飞书回调处理 |
| `/api/auth/me` | GET | 获取当前用户信息 |
| `/api/auth/logout` | POST | 退出登录 |

## 使用示例

### 前端权限检查
```typescript
import { useAuthStore } from '@/stores/auth'

const authStore = useAuthStore()

// 检查单个权限
if (authStore.hasPermission('users.manage')) {
  // 显示用户管理功能
}

// 检查多个权限（满足任一）
if (authStore.hasAnyPermission(['users.manage', 'users.view'])) {
  // 显示用户相关功能
}

// 获取当前用户
const currentUser = authStore.user
```

### 后端路由保护
```typescript
import { requireAuth, requireAdmin } from './middleware/auth'

// 需要登录
router.get('/api/protected', requireAuth, handler)

// 需要管理员权限
router.post('/api/admin-only', requireAdmin, handler)
```

## 注意事项

1. **角色分配**：
   - 用户首次登录时根据 `admins.json` 配置分配角色
   - 已存在用户登录时会更新角色（如果配置有变化）

2. **权限继承**：
   - 角色权限具有层级关系
   - 高级角色自动拥有低级角色的所有权限

3. **会话安全**：
   - 生产环境务必设置强密码的 `SESSION_SECRET`
   - 建议启用 HTTPS
