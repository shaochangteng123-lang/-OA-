# 13-部门管理模块

## 模块概述

部门管理模块负责管理政府部门信息，为事件库提供部门关联数据。系统中的事件可以关联到具体的政府部门，方便项目管理时了解需要对接的部门。

## 主要功能

### 1. 部门信息管理
- 部门的创建、编辑、删除
- 部门全称和简称管理
- 部门网站链接管理
- 部门排序设置
- 部门状态管理（启用/停用）

### 2. 部门搜索
- 按部门全称搜索
- 按部门简称搜索
- 实时搜索过滤

### 3. 部门关联
- 事件库与部门关联
- 删除前检查关联关系
- 防止误删除有关联的部门

## 数据模型

### 政府部门 (Department)

```typescript
interface Department {
  id: string                    // 部门ID
  fullName: string              // 部门全称
  shortNames: string[]          // 简称列表（可以有多个简称）
  websiteUrl?: string           // 部门网站地址
  sortOrder: number             // 排序顺序
  isActive: boolean             // 是否启用
  createdAt: string            // 创建时间
  updatedAt: string            // 更新时间
}
```

**字段说明**:
- `fullName`: 部门的完整官方名称，必须唯一
- `shortNames`: 部门的简称数组，可以包含多个常用简称
- `websiteUrl`: 部门官方网站地址，可选
- `sortOrder`: 用于控制部门在列表中的显示顺序，数值越小越靠前
- `isActive`: 控制部门是否在系统中启用，停用的部门不会在列表中显示

## API 接口

| 接口 | 方法 | 权限 | 说明 |
|------|------|------|------|
| `/api/departments` | GET | 登录用户 | 获取部门列表，支持搜索 |
| `/api/departments/:id` | GET | 登录用户 | 获取单个部门详情 |
| `/api/departments` | POST | 管理员 | 创建新部门 |
| `/api/departments/:id` | PUT | 管理员 | 更新部门信息 |
| `/api/departments/:id` | DELETE | 管理员 | 删除部门 |

## 页面功能详解

### 部门管理页面

**路由**: `/departments`

**页面布局**:
- 页面头部：标题 + 刷新按钮 + 新增按钮
- 搜索栏：支持按部门名称搜索
- 部门表格：展示所有部门信息

**表格列**:
- 部门全称
- 简称（多个简称以标签形式展示）
- 网站（可点击链接访问）
- 排序
- 状态（启用/停用）
- 操作（编辑/删除）

**新增/编辑对话框**:
- 全称（必填）
- 简称（必填，支持添加多个）
- 网站（可选）
- 排序（数字输入）
- 状态（仅编辑时显示，开关切换）

### 功能特点

1. **多简称支持**:
   - 一个部门可以有多个简称
   - 使用 Element Plus 的 Select 组件，支持输入并回车添加
   - 简称以标签形式展示

2. **搜索功能**:
   - 支持按全称搜索
   - 支持按简称搜索
   - 回车键触发搜索
   - 清空按钮快速重置

3. **网站链接**:
   - 如果部门有网站，显示可点击的链接
   - 点击在新标签页打开
   - 没有网站显示"无"

4. **状态管理**:
   - 启用状态显示绿色标签
   - 停用状态显示红色标签
   - 停用的部门不会在列表中显示

5. **删除保护**:
   - 删除前检查是否有事件关联
   - 如果有关联事件，禁止删除并提示关联数量
   - 需要先解除关联才能删除

## 数据库表结构

### government_departments 表

```sql
CREATE TABLE government_departments (
  id TEXT PRIMARY KEY,
  full_name TEXT NOT NULL UNIQUE,
  short_names TEXT NOT NULL,
  website_url TEXT,
  sort_order INTEGER NOT NULL DEFAULT 0,
  is_active INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

-- 索引
CREATE INDEX idx_government_departments_full_name ON government_departments(full_name);
CREATE INDEX idx_government_departments_short_names ON government_departments(short_names);
CREATE INDEX idx_government_departments_is_active ON government_departments(is_active);
CREATE INDEX idx_government_departments_sort_order ON government_departments(sort_order);
```

**字段说明**:
- `id`: 主键，使用 nanoid 生成
- `full_name`: 部门全称，唯一约束
- `short_names`: 简称列表，以逗号分隔存储
- `website_url`: 网站地址，可为空
- `sort_order`: 排序顺序，默认为 0
- `is_active`: 是否启用，1 为启用，0 为停用
- `created_at`: 创建时间，ISO 8601 格式
- `updated_at`: 更新时间，ISO 8601 格式

## 与其他模块的关联

### 事件库模块
- 事件库中的事件可以关联到政府部门
- 通过 `department_id` 字段建立外键关系
- 删除部门前会检查是否有事件关联

```sql
-- event_library 表中的外键
FOREIGN KEY (department_id) REFERENCES government_departments(id)
```

### 日历模块
- 日程事件可以通过事件库间接关联到部门
- 方便了解某个事件需要对接哪个政府部门

## 权限控制

### 查看权限
- **所有登录用户**: 可以查看部门列表和详情
- 用于在创建事件时选择关联部门

### 管理权限
- **管理员**: 可以创建、编辑、删除部门
- 使用 `requireAdmin` 中间件保护

## 业务场景

### 典型使用流程

1. **初始化部门数据**:
   - 系统管理员添加常用的政府部门
   - 设置部门全称、简称、网站
   - 按重要性设置排序

2. **创建事件时关联部门**:
   - 在事件库中创建事件
   - 选择需要对接的政府部门
   - 系统记录部门关联关系

3. **项目管理中使用**:
   - 查看项目日程时，可以看到事件关联的部门
   - 快速了解需要对接哪些政府部门
   - 点击部门网站链接查看相关信息

## 注意事项

1. **唯一性约束**: 部门全称必须唯一，创建或更新时会检查
2. **简称格式**: 简称以逗号分隔存储在数据库中，前端自动处理转换
3. **删除限制**: 有关联事件的部门不能删除，需要先解除关联
4. **状态过滤**: 查询时默认只返回启用状态的部门
5. **搜索范围**: 搜索功能同时匹配全称和简称
