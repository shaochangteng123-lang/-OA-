# 账号密码认证模块

## 模块概述

账号密码认证模块提供了系统的用户认证和授权功能，取代了原有的飞书OAuth认证方式。本模块支持：

- 账号密码登录
- 密码修改
- 管理员创建/管理用户账号
- 基础审批功能

## 数据模型

### 用户表 (users)

```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  username TEXT UNIQUE,           -- 登录用户名
  password_hash TEXT,             -- 密码哈希（bcrypt）
  name TEXT NOT NULL,             -- 显示名称
  email TEXT,
  mobile TEXT,
  avatar_url TEXT,
  role TEXT NOT NULL DEFAULT 'user',  -- super_admin, admin, user, guest
  status TEXT NOT NULL DEFAULT 'active',  -- active, inactive
  department TEXT,
  position TEXT,
  force_change_password BOOLEAN NOT NULL DEFAULT false,  -- 是否需要强制修改密码
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  last_login_at TEXT
);
```

### 审批流程表 (approval_flows)

```sql
CREATE TABLE approval_flows (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,             -- 流程名称
  type TEXT NOT NULL,             -- 审批类型
  steps_json TEXT NOT NULL,       -- 审批步骤配置（JSON）
  is_active INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### 审批实例表 (approval_instances)

```sql
CREATE TABLE approval_instances (
  id TEXT PRIMARY KEY,
  flow_id TEXT,
  type TEXT NOT NULL,
  target_id TEXT NOT NULL,
  target_type TEXT NOT NULL,
  applicant_id TEXT NOT NULL,
  current_step INTEGER NOT NULL DEFAULT 1,
  status TEXT NOT NULL DEFAULT 'pending',  -- pending, approved, rejected, cancelled
  submit_time TEXT NOT NULL,
  complete_time TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

### 审批记录表 (approval_records)

```sql
CREATE TABLE approval_records (
  id TEXT PRIMARY KEY,
  instance_id TEXT NOT NULL,
  step INTEGER NOT NULL,
  approver_id TEXT NOT NULL,
  action TEXT NOT NULL,           -- approve, reject, comment
  comment TEXT,
  action_time TEXT NOT NULL
);
```

## API 接口

### 认证接口

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/auth/login` | POST | 账号密码登录 | 公开 |
| `/api/auth/logout` | POST | 登出 | 已登录 |
| `/api/auth/user` | GET | 获取当前用户 | 已登录 |
| `/api/auth/session` | GET | 检查会话状态 | 公开 |
| `/api/auth/change-password` | POST | 修改密码 | 已登录 |

### 用户管理接口

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/users` | GET | 获取用户列表 | admin+ |
| `/api/users` | POST | 更新用户信息 | admin+ |
| `/api/users/create` | POST | 创建新用户 | super_admin |
| `/api/users/:id` | DELETE | 删除用户 | super_admin |
| `/api/users/:id/reset-password` | POST | 重置用户密码 | super_admin |

| 接口 | 方法 | 说明 | 权限 |
|------|------|------|------|
| `/api/approval/pending` | GET | 获取待审批列表 | admin+ |

| `/api/approval/processed` | GET | 获取已处理列表 | admin+ |
| `/api/approval/:id` | GET | 获取审批详情 | 已登录 |

| `/api/approval/:id/approve` | POST | 通过审批 | admin+ |

| `/api/approval/:id/cancel` | POST | 撤销审批 | 申请人 |

## 使用说明

### 1. 创建初始管理员

首次部署系统时，需要通过命令行创建管理员账号：

```bash
npm run create:admin
```

按照提示输入：
- 用户名（仅字母、数字、下划线，至少3个字符）
- 密码（至少6个字符）
- 显示名称

### 2. 登录系统

访问登录页面，输入用户名和密码即可登录。

### 3. 管理用户

超级管理员可以在"用户管理"页面：
- 创建新用户
- 编辑用户信息（角色、状态、部门等）
- 重置用户密码
- 删除用户

### 4. 修改密码

用户可以在"个人设置"页面修改自己的密码。

### 5. 审批功能

- 员工提交审批请求（工作日志、报销单等）
- 管理员在"审批中心"处理审批
- 支持通过/驳回/撤销操作

## 安全措施

### 密码安全

- 使用 bcrypt 算法进行密码哈希（salt rounds = 10）
- 密码最小长度 6 个字符
- 密码最大长度 128 个字符
- 不存储明文密码

### 会话管理

- 使用 PostgreSQL 存储会话
- 会话有效期 7 天
- HttpOnly Cookie
- SameSite = 'lax'

### 权限控制

- 四级角色：super_admin > admin > user > guest
- 敏感操作（创建/删除用户、重置密码）仅 super_admin 可用
- 审批操作需要 admin 及以上权限
