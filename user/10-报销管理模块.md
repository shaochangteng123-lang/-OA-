# 10 - 报销管理模块

## 模块概述

报销管理模块负责处理员工的日常报销申请，包括基础报销、大额报销和商务报销。支持发票上传、OCR自动识别、PDF压缩等功能。

## 主要功能

### 1. 基础报销
- 报销单创建与编辑
- 报销类型选择（行政办公、三方服务、业务提升）
- 发票上传与OCR识别
- 报销金额和日期自动填充
- 草稿保存与提交审批

### 2. 发票处理
- PDF格式验证
- 文件大小限制（5MB）
- 超大文件自动压缩
- 发票OCR自动识别
- 发票文件存储管理

### 3. OCR识别
- 百度OCR集成（主要方案）
- 腾讯云OCR集成（备用方案）
- 自动识别报销金额（价税合计）
- 自动识别开票日期
- 自动识别发票号码
- 识别结果自动填充后不可手动修改

### 上传发票接口

**请求**：
```typescript
POST /api/reimbursement/upload-invoice
Content-Type: multipart/form-data

{
  invoice: File // PDF文件
}
```

**响应**：
```typescript
{
  success: true,
  message: '发票上传成功',
  data: {
    fileName: 'invoice-1234567890-发票.pdf',
    filePath: '/uploads/invoices/invoice-1234567890-发票.pdf',
    compressed: false, // 是否经过压缩
    ocrResult: {
      amount: 350.50,
      date: '2026-01-23',
      invoiceNumber: '12345678',
      seller: '北京某某公司'
    }
  }
}
```

### PDF压缩接口

**请求**：
```typescript

Content-Type: multipart/form-data

}
```

**响应**：

## OCR配置

### 本地 OCR 识别

系统使用**本地 OCR** 进行发票识别，具有以下特点：

- ✅ **数据安全**：所有识别在本地完成，发票数据不会上传到外部服务器
- ✅ **无需配置**：开箱即用，无需申请任何 API 密钥
- ✅ **零成本**：完全免费，无调用次数限制
- ⚠️ **准确率**：识别准确率可能低于云端服务，建议人工核对

### OCR 技术栈

- **Tesseract.js**：开源 OCR 引擎，支持中英文识别
- **PDF 解析**：使用 pdf-lib 进行 PDF 文件处理
- **正则匹配**：智能提取发票关键信息（金额、日期、发票号码等）

### 识别流程

1. 用户上传 PDF 发票
2. 系统在本地解析 PDF 文件
3. 使用正则表达式提取关键信息
4. 返回识别结果供用户确认
5. 用户可手动修改识别结果

### 注意事项

- 本地 OCR 适用于标准格式的电子发票
- 对于扫描件或特殊格式发票，建议手动填写
- 系统会自动标记识别结果的可信度
- 所有识别结果都需要用户最终确认

### 文件上传处理

```typescript
const handleFileChange = async (file: any, fileList: any[]) => {
  // 验证文件格式 - 严格检查
  const fileName = file.name?.toLowerCase() || ''
  const isImageFile = fileName.endsWith('.jpg') || fileName.endsWith('.jpeg') ||
                      fileName.endsWith('.png') || fileName.endsWith('.gif') ||
                      fileName.endsWith('.bmp')

  if (isImageFile) {
    ElMessage.error('请上传正确格式的发票')
    // 移除非PDF文件
    const index = fileList.findIndex(f => f.uid === file.uid)
    if (index > -1) {
      fileList.splice(index, 1)
    }
    return
  }

  if (file.raw && file.raw.type !== 'application/pdf') {
    ElMessage.error('仅支持PDF文件，单个文件不超过5M')
    return
  }

  // 调用后端API进行发票OCR识别
  try {
    ElMessage.info('正在识别发票信息...')

    const formData = new FormData()
    formData.append('invoice', file.raw)

    const response = await fetch('/api/reimbursement/upload-invoice', {
      method: 'POST',
      body: formData,
      credentials: 'include',
    })

    const result = await response.json()

    if (result.success && result.data.ocrResult) {
      const { amount, date, invoiceNumber } = result.data.ocrResult

      // 自动填充识别结果
      formData.amount = amount
      formData.invoiceDate = date
      formData.invoiceNumber = invoiceNumber
      formData.ocrFilled = true // 标记为OCR填充，禁止手动修改

      ElMessage.success('发票识别成功！已自动填充报销金额、开票日期和发票号码')
    }
  } catch (error) {
    ElMessage.error('发票识别失败，请手动填写信息')
  }
}
```

## 后端实现

### PDF压缩功能

```typescript
async function compressPDF(inputPath: string, outputPath: string): Promise<void> {
  const existingPdfBytes = fs.readFileSync(inputPath)
  const pdfDoc = await PDFDocument.load(existingPdfBytes)

  // 移除元数据以减小文件大小
  pdfDoc.setTitle('')
  pdfDoc.setAuthor('')
  pdfDoc.setSubject('')
  pdfDoc.setKeywords([])
  pdfDoc.setProducer('')
  pdfDoc.setCreator('')

  // 保存压缩后的PDF
  const pdfBytes = await pdfDoc.save({
    useObjectStreams: false,
  })

  fs.writeFileSync(outputPath, pdfBytes)
}
```

### 百度OCR识别

```typescript
async function recognizeInvoiceWithBaidu(filePath: string) {
  // 1. 获取access_token
  const tokenResponse = await axios.post(
    'https://aip.baidubce.com/oauth/2.0/token',
    null,
    {
      params: {
        grant_type: 'client_credentials',
        client_id: process.env.BAIDU_OCR_API_KEY,
        client_secret: process.env.BAIDU_OCR_SECRET_KEY,
      },
    }
  )

  const accessToken = tokenResponse.data.access_token

  // 2. 读取PDF文件并转换为base64
  const fileBuffer = fs.readFileSync(filePath)
  const base64Image = fileBuffer.toString('base64')

  // 3. 调用增值税发票识别API
  const ocrResponse = await axios.post(
    `https://aip.baidubce.com/rest/2.0/ocr/v1/vat_invoice?access_token=${accessToken}`,
    `pdf_file=${encodeURIComponent(base64Image)}&pdf_file_num=1`,
    {
      headers: {
        'Content-Type': 'application/x-www-form-urlencoded',
      },
    }
  )

  const result = ocrResponse.data.words_result

  // 4. 解析识别结果
  return {
    amount: parseFloat(result.AmountInFiguers || result.TotalAmount || '0'),
    date: result.InvoiceDate || new Date().toISOString().split('T')[0],
    invoiceNumber: result.InvoiceNum || result.InvoiceCode,
    seller: result.SellerName,
  }
}
```

## 注意事项

1. **文件格式**：
   - 仅支持PDF格式
   - JPG、PNG等图片格式会提示"请上传正确格式的发票"
   - 前端和后端都进行格式验证

2. **文件大小**：
   - 单个文件最大5MB
   - 超过5MB会提示错误，不允许上传

3. **OCR识别**：
   - 需要配置百度OCR或腾讯云OCR
   - 未配置时使用模拟数据
   - 识别字段包括：报销金额、开票日期、发票号码
   - 识别结果自动填充后不可手动修改

4. **报销月份**：
   - 自动显示当前月份（格式：YYYY年MM月）
   - 显示在报销类型上方

5. **发票预览**：
   - 上传后显示PDF文档图标
   - 双击缩略图可在新窗口预览发票
   - 双击时不会选中文字

6. **安全性**：
   - 所有接口需要登录认证
   - 文件上传使用multer中间件
   - 防止路径遍历攻击

7. **错误处理**：
   - 文件上传失败自动清理临时文件
   - OCR识别失败提示用户重新上传
   - 格式错误自动移除文件
