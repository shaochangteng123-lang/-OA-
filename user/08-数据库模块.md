### 1. Users（用户表）
```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT,
  feishu_user_id TEXT UNIQUE,
  feishu_username TEXT,
  avatar_url TEXT,
  role TEXT DEFAULT 'user',
  is_active BOOLEAN DEFAULT 1,
  last_login_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP

CREATE INDEX idx_users_feishu_user_id ON users(feishu_user_id);
```

### 2. Projects（项目表）
```sql
CREATE TABLE IF NOT EXISTS projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  project_code TEXT UNIQUE,
  type TEXT,
  status TEXT DEFAULT 'PLANNING',
  district TEXT,
  description TEXT,
  start_date DATE,
  end_date DATE,
  contact_person TEXT,

  budget REAL,
  actual_cost REAL,
  created_by INTEGER NOT NULL,

  FOREIGN KEY (created_by) REFERENCES users(id)

CREATE INDEX idx_projects_status ON projects(status);
CREATE INDEX idx_projects_district ON projects(district);
CREATE INDEX idx_projects_created_by ON projects(created_by);
```

## 数据库操作

### 基础查询
```typescript
import { db } from './db'

// 查询单条
const user = db.prepare('SELECT * FROM users WHERE id = ?').get(userId)

// 查询多条
const users = db.prepare('SELECT * FROM users WHERE role = ?').all('admin')

// 插入
const result = db.prepare(`
  INSERT INTO users (name, email, role)
  VALUES (?, ?, ?)
`).run(name, email, role)

const newUserId = result.lastInsertRowid

// 更新
db.prepare('UPDATE users SET role = ? WHERE id = ?').run(role, userId)

// 删除
db.prepare('DELETE FROM users WHERE id = ?').run(userId)
```

### 事务处理
```typescript
const createProject = db.transaction((projectData) => {
  // 插入项目
  const project = db.prepare(`
    INSERT INTO projects (name, type, created_by)
    VALUES (?, ?, ?)
  `).run(projectData.name, projectData.type, projectData.userId)

  // 插入关联的事件
  const eventStmt = db.prepare(`
    INSERT INTO calendar_events (title, project_id, created_by)
    VALUES (?, ?, ?)
  `)

  projectData.events.forEach(event => {
    eventStmt.run(event.title, project.lastInsertRowid, projectData.userId)
  })

  return project.lastInsertRowid
})

// 执行事务
try {
  const projectId = createProject(projectData)
} catch (error) {
  // 事务自动回滚
  console.error('创建项目失败:', error)
}
```

### 预编译语句
```typescript
// 提高性能，避免重复编译
const getUserStmt = db.prepare('SELECT * FROM users WHERE id = ?')

const user1 = getUserStmt.get(1)
const user2 = getUserStmt.get(2)
```

## 性能优化

### 1. 索引优化
```sql

  ON worklogs(user_id, log_date);

-- 覆盖索引

  ON users(role, is_active);
```

### 2. 查询优化
```typescript
// 使用参数化查询
const stmt = db.prepare('SELECT * FROM users WHERE role = ?')
const admins = stmt.all('admin')

// 限制返回字段
const users = db.prepare(`
  SELECT id, name, role FROM users

// 分页查询

const users = db.prepare(`
  SELECT * FROM users
```

## 数据完整性

### 外键约束
```sql
-- PostgreSQL 默认启用外键约束，无需 PRAGMA

-- 定义外键
FOREIGN KEY (user_id) REFERENCES users(id)
  ON DELETE CASCADE
```

### 唯一约束
```sql
-- 单字段唯一
CREATE TABLE users (
  email TEXT UNIQUE
);

-- 复合唯一
CREATE TABLE worklogs (
  UNIQUE(user_id, log_date)
);
```

### 检查约束
```sql
CREATE TABLE projects (
  budget REAL CHECK(budget >= 0),
  status TEXT CHECK(status IN ('PLANNING', 'ACTIVE', 'COMPLETED'))
);
```

## 注意事项

1. **并发控制**：
   - PostgreSQL 支持并发读写
   - 使用 WAL 模式提高并发性能
   - 长事务会阻塞写操作

2. **数据类型**：
   - PostgreSQL 是强类型
   - 建议在应用层严格验证
   - 使用 TEXT 存储 JSON 数据

3. **备份策略**：
   - 定期备份数据库文件
   - 重要操作前手动备份
   - 测试恢复流程

4. **性能监控**：
   - 分析慢查询
   - 定期 VACUUM 清理
   - 监控数据库文件大小
