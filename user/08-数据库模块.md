# 08 - 数据库模块

## 模块概述

数据库模块负责 PostgreSQL 数据库的初始化、表结构定义、数据迁移和种子数据管理。

## 主要功能

### 1. 数据库初始化
- PostgreSQL 数据库连接
- 表结构自动创建
- 索引和约束设置
- 默认数据填充

### 2. 数据表管理
- 用户表
- 项目表
- 工作日志表
- 日历事件表
- 事件库表
- 节假日表
- 会话表

### 3. 数据迁移
- 数据库版本管理
- 结构变更迁移
- 数据迁移脚本

### 4. 种子数据
- 初始化数据填充
- 测试数据生成
- 开发环境数据

### 1. Users（用户表）
```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT,
  feishu_user_id TEXT UNIQUE,
  feishu_username TEXT,
  avatar_url TEXT,
  role TEXT DEFAULT 'user',
  is_active BOOLEAN DEFAULT 1,

  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_feishu_user_id ON users(feishu_user_id);
```

### 2. Projects（项目表）
```sql
CREATE TABLE IF NOT EXISTS projects (
```

### 9. Human Resources（人事流程表）
```sql
CREATE TABLE IF NOT EXISTS probation_confirmations (
  id TEXT PRIMARY KEY,
  employee_id TEXT NOT NULL REFERENCES employee_profiles(id) ON DELETE CASCADE,
  hire_date TEXT NOT NULL,
  probation_end_date TEXT NOT NULL,
  status TEXT DEFAULT 'pending',
  submit_time TEXT,
  approve_time TEXT,
  approver_id TEXT REFERENCES users(id),
  approver_comment TEXT,
  application_comment TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS probation_documents (
  id TEXT PRIMARY KEY,

  document_type TEXT NOT NULL,
  file_name TEXT NOT NULL,
  file_path TEXT NOT NULL,
  file_size INTEGER,
  mime_type TEXT,
  uploaded_by TEXT NOT NULL REFERENCES users(id),
  uploaded_by_name TEXT,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS probation_templates (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  file_name TEXT NOT NULL,
  file_path TEXT NOT NULL,
  file_size INTEGER,
  mime_type TEXT,
  uploaded_by TEXT NOT NULL REFERENCES users(id),
  uploaded_by_name TEXT,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS resignation_requests (
  id TEXT PRIMARY KEY,
  employee_id TEXT NOT NULL REFERENCES employee_profiles(id) ON DELETE CASCADE,
  employee_user_id TEXT NOT NULL REFERENCES users(id),
  handover_user_id TEXT NOT NULL REFERENCES users(id),
  handover_name TEXT,
  resign_type TEXT NOT NULL,
  resign_date TEXT NOT NULL,
  reason TEXT,
  status TEXT NOT NULL DEFAULT 'draft',
  employee_confirm_time TEXT,
  handover_confirm_time TEXT,
  submit_time TEXT,
  approve_time TEXT,
  approver_id TEXT REFERENCES users(id),
  approver_comment TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  UNIQUE(employee_id)
);

CREATE TABLE IF NOT EXISTS resignation_documents (
  id TEXT PRIMARY KEY,
  request_id TEXT NOT NULL REFERENCES resignation_requests(id) ON DELETE CASCADE,
  document_type TEXT NOT NULL,
  uploader_role TEXT NOT NULL,
  file_name TEXT NOT NULL,
  file_path TEXT NOT NULL,
  file_size INTEGER,
  mime_type TEXT,
  uploaded_by TEXT NOT NULL REFERENCES users(id),
  uploaded_by_name TEXT,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS resignation_templates (
  id TEXT PRIMARY KEY,
  template_type TEXT NOT NULL,
  name TEXT NOT NULL,
  file_name TEXT NOT NULL,
  file_path TEXT NOT NULL,
  file_size INTEGER,
  mime_type TEXT,
  uploaded_by TEXT NOT NULL REFERENCES users(id),
  uploaded_by_name TEXT,
  created_at TEXT NOT NULL
);
```

**说明：**
- `resignation_requests` 保存离职主流程，支持草稿、提交、交接确认、管理员审批等状态。
- `resignation_documents` 按申请表、双方交接单、补充材料和上传角色区分离职材料，也是员工详情“离职档案”Tab 的附件数据来源。
- `resignation_templates` 由管理员维护固定模板，除申请表与交接单外，还支持终止/解除劳动关系证明、固定资产交接单、离职经济补偿协议书、离职其他费用结算约定、合伙人离任分红结算模板。
- `resignation_requests.employee_id` 采用唯一约束，便于按员工维度聚合查询离职档案；若无离职申请记录，可结合 `employee_profiles.employment_status = 'resigned'` 做前端展示兜底。

## 数据库操作

### 基础查询
```typescript
import { db } from './db'

// 查询单条
const user = db.prepare('SELECT * FROM users WHERE id = ?').get(userId)

// 查询多条
const users = db.prepare('SELECT * FROM users WHERE role = ?').all('admin')

// 插入
const result = db.prepare(`
  INSERT INTO users (name, email, role)
  VALUES (?, ?, ?)
`).run(name, email, role)

const newUserId = result.lastInsertRowid

// 更新
db.prepare('UPDATE users SET role = ? WHERE id = ?').run(role, userId)

// 删除
db.prepare('DELETE FROM users WHERE id = ?').run(userId)
```

### 事务处理
```typescript
const createProject = db.transaction((projectData) => {
  // 插入项目
  const project = db.prepare(`
    INSERT INTO projects (name, type, created_by)
    VALUES (?, ?, ?)
  `).run(projectData.name, projectData.type, projectData.userId)

  // 插入关联的事件
  const eventStmt = db.prepare(`
    INSERT INTO calendar_events (title, project_id, created_by)
    VALUES (?, ?, ?)
  `)

  projectData.events.forEach(event => {
    eventStmt.run(event.title, project.lastInsertRowid, projectData.userId)
  })

  return project.lastInsertRowid
})

// 执行事务
try {
  const projectId = createProject(projectData)
} catch (error) {
  // 事务自动回滚
  console.error('创建项目失败:', error)
}
```

### 预编译语句
```typescript
// 提高性能，避免重复编译
const getUserStmt = db.prepare('SELECT * FROM users WHERE id = ?')

const user1 = getUserStmt.get(1)
const user2 = getUserStmt.get(2)
```

## 注意事项

1. **并发控制**：
   - PostgreSQL 支持并发读写
   - 使用 WAL 模式提高并发性能
   - 长事务会阻塞写操作

2. **数据类型**：
   - PostgreSQL 是强类型
   - 建议在应用层严格验证
   - 使用 TEXT 存储 JSON 数据

3. **备份策略**：
   - 定期备份数据库文件
   - 重要操作前手动备份
   - 测试恢复流程

4. **性能监控**：
   - 分析慢查询
   - 定期 VACUUM 清理
   - 监控数据库文件大小
