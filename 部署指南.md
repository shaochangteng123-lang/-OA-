# YuliLog 工作日志系统 - 生产环境部署指南

## 📋 部署方案概述

本系统支持在同一台服务器上同时运行**生产环境**和**开发环境**，两者完全隔离：

- **生产环境**：端口 8899，使用优化的 Docker 镜像，数据库独立
- **开发环境**：端口 8900，支持热重载，数据库独立

## 🚀 快速部署（推荐方案）

### 方案一：生产环境 + 开发环境同时运行

#### 1. 启动生产环境

```bash
# 使用生产配置启动
docker compose -f docker-compose.prod.yml up -d --build

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 访问地址
http://localhost:8899
```

#### 2. 启动开发环境（端口 8900）

```bash
# 使用开发配置启动
docker compose -f docker-compose.dev.yml up -d

# 查看日志
docker compose -f docker-compose.dev.yml logs -f

# 访问地址
http://localhost:8900  # 前端
http://localhost:3001  # 后端 API
```

#### 3. 开发完成后更新生产环境

```bash
# 停止生产环境
docker compose -f docker-compose.prod.yml down

# 重新构建并启动
docker compose -f docker-compose.prod.yml up -d --build
```

## 📝 详细操作步骤

### 第一步：准备环境变量

#### 生产环境配置（.env.production）

```bash
# 已存在，确认配置正确
cat .env.production
```

关键配置项：
- `NODE_ENV=production`
- `PORT=8899`
- `DATABASE_URL=postgresql://postgres:postgres@postgres:5432/yulilog_worklog`
- `SESSION_SECRET`（生产环境务必修改为强密码）

#### 开发环境配置（.env）

```bash
# 已存在，用于开发环境
cat .env
```

### 第二步：创建管理员账号（首次部署）

```bash
# 在生产环境容器中创建管理员
docker compose -f docker-compose.prod.yml exec yulilog npm run create:admin

# 或者在启动前本地创建
npm run create:admin
```

### 第三步：启动服务

#### 仅启动生产环境

```bash
docker compose -f docker-compose.prod.yml up -d --build
```

#### 同时启动生产和开发环境

```bash
# 启动生产环境
docker compose -f docker-compose.prod.yml up -d --build

# 启动开发环境（需要先修改端口）
docker compose -f docker-compose.dev.yml up -d
```

## 🔧 端口配置说明

### 当前端口分配

| 环境 | 前端端口 | 后端端口 | 数据库端口 | 容器名称 |
|------|---------|---------|-----------|----------|
| 生产环境 | 8899 | - | - | yulilog-prod |
| 开发环境 | 8899 | 3000 | 5432 | yulilog-worklog-dev |

### ⚠️ 端口冲突问题

如果要同时运行两个环境，需要修改开发环境端口避免冲突。

```bash
# 生产环境日志
```

## 🗄️ 数据管理

### 数据库备份

```bash
# 备份生产数据库
docker compose -f docker-compose.prod.yml exec postgres pg_dump -U postgres yulilog_worklog > backup_$(date +%Y%m%d).sql
```

### 数据库恢复

```bash
# 恢复数据库
docker compose -f docker-compose.prod.yml exec -T postgres psql -U postgres yulilog_worklog < backup_20260330.sql
```

### 数据持久化

数据存储在 Docker 卷中：
- 生产环境：`postgres_data_prod`
- 开发环境：`postgres_data_dev`
- 上传文件：`./uploads` 目录

---

## ⚠️ 注意事项

1. **环境隔离**：生产和开发环境使用独立的数据库，数据不互通
2. **端口冲突**：同时运行需要修改开发环境端口
3. **数据备份**：定期备份生产数据库
4. **密钥安全**：生产环境的 `SESSION_SECRET` 必须修改为强密码
5. **日志监控**：定期查看日志，及时发现问题

## 🐛 故障排查

### 容器无法启动

```bash
# 查看详细日志
docker compose -f docker-compose.prod.yml logs

# 检查端口占用
lsof -i :8899
```

### 数据库连接失败

```bash
# 检查数据库状态
docker compose -f docker-compose.prod.yml exec postgres pg_isready -U postgres

# 查看数据库日志
docker compose -f docker-compose.prod.yml logs postgres
```

### 前端无法访问

```bash
# 检查容器健康状态
docker compose -f docker-compose.prod.yml ps

# 测试健康检查接口
curl http://localhost:8899/api/health
```
