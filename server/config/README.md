# 管理员配置说明

## 配置文件：admins.json

此配置文件用于管理系统的超级管理员和普通管理员。

### 配置格式

```json
{
  "superAdmins": [
    {
      "name": "用户512390",
      "comment": "主要超级管理员"
    }
  ],
  "admins": [
    {
      "name": "张三",
      "comment": "部门管理员"
    }
  ],
  "description": "管理员配置文件说明"
}
```

### 字段说明

- **superAdmins**: 超级管理员列表
  - `name`: 飞书用户名（必须与飞书账号显示名称完全一致）
  - `comment`: 备注信息（可选）

- **admins**: 普通管理员列表
  - `name`: 飞书用户名（必须与飞书账号显示名称完全一致）
  - `comment`: 备注信息（可选）

### 工作原理

1. **首次登录**：当用户首次通过飞书登录时，系统会检查其用户名是否在配置文件中
   - 如果在 `superAdmins` 列表中 → 角色设为 `super_admin`
   - 如果在 `admins` 列表中 → 角色设为 `admin`
   - 否则 → 角色设为 `user`（普通用户）

2. **后续登录**：每次登录时，系统都会检查配置文件并自动更新用户角色
   - 如果配置文件中的角色有变化，会自动同步到数据库
   - 例如：将普通用户添加到 `superAdmins` 列表后，下次登录即可获得超级管理员权限

3. **降级处理**：从配置文件中移除用户后，下次登录时角色会自动降为普通用户

### 注意事项

1. **用户名必须完全匹配**：配置文件中的 `name` 必须与飞书账号的显示名称完全一致（包括大小写）

2. **修改配置后无需重启**：配置文件在服务器启动时加载，如需更新配置需要重启Docker容器：
   ```bash
   docker-compose down && npm run docker:build && npm run docker:run
   ```

3. **配置文件位置**：`server/config/admins.json`

4. **备份建议**：修改配置前建议备份当前配置文件

### 角色权限说明

| 角色 | 权限 |
|-----|------|
| super_admin | 所有权限（用户管理、角色分配、系统设置等） |
| admin | 管理员权限（查看所有数据、管理事件库等，但不能分配超级管理员角色） |
| user | 普通用户权限（只能管理自己的数据） |
| guest | 访客权限（只读） |

### 示例场景

#### 场景1：添加新的超级管理员

```json
{
  "superAdmins": [
    {
      "name": "用户512390",
      "comment": "主要超级管理员"
    },
    {
      "name": "李四",
      "comment": "备用超级管理员"
    }
  ]
}
```

用户"李四"下次登录时会自动获得超级管理员权限。

#### 场景2：将超级管理员降为普通管理员

```json
{
  "superAdmins": [
    {
      "name": "用户512390",
      "comment": "主要超级管理员"
    }
  ],
  "admins": [
    {
      "name": "李四",
      "comment": "从超级管理员降级"
    }
  ]
}
```

用户"李四"下次登录时会自动降为普通管理员。

#### 场景3：撤销管理员权限

```json
{
  "superAdmins": [
    {
      "name": "用户512390",
      "comment": "主要超级管理员"
    }
  ],
  "admins": []
}
```

其他所有用户下次登录时会自动变为普通用户。
